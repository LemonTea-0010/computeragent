```markdown
# 项目：基于多智能体协同的舆情分析系统 — 前端搭建

## 一、项目概述

我正在做一个计算机设计大赛的项目，主题是「基于多智能体协同的舆情分析系统」。
系统融合了时空分析和网络安全两大专业特色。
现在需要你帮我先把前端框架和所有页面搭建出来，后端API后续对接，
当前阶段前端用Mock数据/静态数据填充即可。

## 二、技术栈要求

- 框架：Vue3 + TypeScript
- 构建工具：Vite
- UI组件库：Element Plus
- 可视化：ECharts 5
- 路由：Vue Router 4
- 状态管理：Pinia
- CSS方案：SCSS + 适当使用 TailwindCSS
- HTTP请求：Axios（先封装好，后续对接后端）
- 地图：ECharts 地图（中国地图 + 世界地图）
```

```
## 四、各页面详细要求

### 4.0 全局主题风格
- 整体采用 **深色科技风** 主题（深蓝/深灰底色，科技蓝 #409EFF 和青色 #00F0FF 为主色调）
- 卡片组件有微弱的发光边框效果
- 数字使用 Digital 风格字体或加粗醒目显示
- 页面切换有平滑过渡动画

### 4.1 登录页 (Login/index.vue)
- 科技感背景（粒子动画或渐变动态背景）
- 系统名称：「舆情智析 — 基于多智能体协同的舆情分析系统」
- 用户名 + 密码登录表单
- 登录按钮，点击后跳转到 Dashboard

### 4.2 首页 — 态势总览大屏 (Dashboard/index.vue) ⭐ 重点页面
这是展示的核心页面，类似指挥大屏，布局参考如下：

```
┌──────────────────────────────────────────────────────┐
│  舆情智析 — 态势感知大屏                 2024-xx-xx    │
├─────────────┬────────────────────┬───────────────────┤
│  左上:       │  中上:              │  右上:            │
│  4个统计卡片  │  中国地图热力图      │  预警信息列表      │
│  ·今日信息量  │  (各省舆情热度)      │  (按颜色分级)      │
│  ·负面占比   │                    │                   │
│  ·活跃话题数  │                    │                   │
│  ·预警数量   │                    │                   │
├─────────────┼────────────────────┼───────────────────┤
│  左下:       │  中下:              │  右下:            │
│  情感分布    │  24h舆情趋势折线图   │  实时热词词云      │
│  (饼图/环形) │  (信息量+情感曲线)   │                   │
│             │                    │                   │
│  平台分布    │                    │  Agent运行状态     │
│  (柱状图)    │                    │  (各Agent在线状态)  │

```

数据使用Mock：
- 统计卡片：今日信息量 125847、负面占比 23.4%、活跃话题 38、预警 5条
- 地图：各省份随机热度值
- 趋势图：24个时间点的折线数据
- 情感饼图：正面45%、中性32%、负面23%
- 热词词云：约30个词（如"AI监管"、"数据安全"、"食品安全"等）
- Agent状态：6个Agent的在线/忙碌状态

### 4.3 数据采集管理 (DataCollection/)
- **任务列表页**：表格展示采集任务（任务名、平台、关键词、状态、创建时间）
- **新建任务**：表单包含——
  - 任务名称
  - 选择平台（多选：微博/知乎/抖音/新闻网站/...）
  - 关键词设置（可添加多个）
  - 采集频率（实时/每小时/每天）
  - 时间范围
  - 启动/暂停按钮

### 4.4 舆情监测 (Monitor/)
- **实时信息流**：瀑布流展示最新信息条目（平台图标、内容摘要、发布时间、情感标签）
- **热点话题排行**：TOP20话题列表，含热度指数、热度趋势迷你图、涉及平台
- **话题详情页**：
  - 话题基本信息
  - 信息量趋势图（时间维度）
  - 地域分布图（空间维度）
  - 情感分析结果
  - 代表性言论列表
  - 相关话题推荐

### 4.5 深度分析 (Analysis/) ⭐ 特色页面
- **情感分析页**：
  - 整体情感分布（饼图）
  - 情感趋势变化（折线图，正/负/中三条线）
  - 细粒度情绪分布（雷达图：愤怒/恐惧/悲伤/喜悦/惊讶/厌恶）
  - 典型正面/负面言论列表

- **观点挖掘页**：
  - 核心观点列表（卡片形式，含持有人数占比）
  - 观点对立面可视化（左右对比布局）
  - 争议焦点排行

- **时空分布分析页（特色）**：
  - 上半部分：中国地图热力图，可切换省份下钻
  - 下半部分左：时间分布图（按小时/天/周的柱状图）
  - 下半部分右：舆情生命周期图（萌芽→爆发→高峰→衰退→消亡阶段标注）
  - 底部：时空关联表格（哪个地区在什么时间段信息量最大）

- **传播者画像页**：
  - KOL影响力排行（头像、昵称、粉丝数、影响力评分）
  - 用户类型分布（普通用户/认证大V/媒体/政府/企业）
  - 活跃度分析图

### 4.6 传播追踪 (Trace/) ⭐ 特色页面
- **传播路径图**：
  - 网络拓扑图（节点=信息源，边=传播关系，用NetworkGraph组件）
  - 可交互：点击节点查看详情
  - 节点大小代表影响力，颜色代表平台类型

- **信息溯源页**：
  - 输入一条信息/URL，展示溯源结果
  - 时间线展示：最早出现→各次转载扩散
  - 首发信息详情卡片

- **时空传播地图（特色）**：
  - 地图上用动态流线展示信息从一个城市传到另一个城市
  - 带时间轴播放器，可以拖动时间轴看不同时刻的传播状态
  - 类似航班流向图的效果

### 4.7 预警中心 (Warning/)
- **预警列表**：
  - 表格：预警时间、事件名称、预警等级（蓝/黄/橙/红 色标）、状态（待处理/已处理）
  - 支持按等级筛选

- **预警规则配置**：
  - 规则列表（规则名、触发条件、通知方式、状态开关）
  - 新建规则表单（信息量阈值、负面比例阈值、关键词触发等）

- **报告生成**：
  - 选择报告类型（日报/周报/专题）
  - 选择时间范围
  - 选择包含模块（情感分析/传播分析/安全分析等复选）
  - 生成预览（简单展示报告结构）
  - 导出按钮（暂时不实现导出功能）

### 4.8 安全态势感知 (Security/) ⭐ 专业特色页面
- **安全态势总览**：
  - 顶部：安全评分仪表盘（当前安全等级评分，如82分）
  - 安全威胁分类统计（虚假信息/恶意行为/敏感信息/意识形态 四类的数量）
  - 最近安全事件列表

- **威胁检测页**：
  - 威胁类型分布（饼图）
  - 威胁等级分布（柱状图）
  - 威胁事件详情表格（时间、类型、来源、严重程度、状态）

- **谣言识别页**：
  - 疑似谣言列表（内容摘要、可信度评分、传播范围、核查状态）
  - 每条可展开查看：AI分析理由、相关事实参考
  - 可信度用进度条展示

- **安全态势地图**：
  - 中国地图 + 全球地图切换
  - 各区域安全等级着色（绿/黄/橙/红）
  - 标注威胁源位置（境外攻击来源用特殊图标）
  - 信息流向连线（从境外到境内的信息攻击路径）

### 4.9 多智能体管理中心 (AgentCenter/) ⭐ 核心创新页面
- **Agent列表**：
  - 卡片式布局展示所有Agent
  - 每个Agent卡片包含：
    - Agent名称和图标（如 🕷️采集Agent、🧠分析Agent、🛡️安全Agent）
    - 当前状态（在线/忙碌/离线，用不同颜色指示灯）
    - 当前执行任务
    - 已完成任务数
    - CPU/内存使用率（进度条）
  - Agent包括：
    1. 调度Agent (Orchestrator)
    2. 微博采集Agent
    3. 新闻采集Agent
    4. 视频采集Agent
    5. 情感分析Agent
    6. 话题分析Agent
    7. 传播追踪Agent
    8. 谣言检测Agent
    9. 安全评估Agent

- **Agent协作日志**：
  - 时间线形式展示Agent间的协作记录
  - 每条日志：时间戳 + 发起Agent + 动作 + 目标Agent + 结果
  - 例如：
    - [10:23:15] 调度Agent → 分配任务 → 微博采集Agent：采集"AI安全"相关信息
    - [10:23:18] 微博采集Agent → 开始执行 → 已采集 0/1000
    - [10:25:33] 微博采集Agent → 任务完成 → 采集1247条，转交分析Agent
    - [10:25:34] 情感分析Agent → 开始分析 → 处理中...
    - [10:28:11] 情感分析Agent → 分析完成 → 正面38%/中性35%/负面27%
    - [10:28:12] 安全Agent → 触发检测 → 发现3条疑似谣言，已标记
  - 支持按Agent类型筛选
```

```
## 六、Mock数据要求

请为每个页面准备合理的Mock数据，放在 mock/ 目录下。数据要看起来真实，比如：
- 舆情内容使用真实感的中文文本（关于AI安全、数据隐私、食品安全等话题）
- 时间数据使用最近7天的日期
- 地域数据覆盖中国各省份
- 数值数据在合理范围内随机

## 七、注意事项

1. 所有ECharts图表请封装为独立组件，通过props传入数据
2. 页面响应式适配，但优先保证 1920x1080 大屏显示效果
3. 代码注释清晰，方便后续迭代
4. Dashboard大屏页面可以做成全屏模式（隐藏侧边栏和顶栏）
5. 侧边栏支持折叠/展开
6. 当前阶段不需要对接真实后端，所有数据从mock文件读取

## 八、开发顺序建议

请按以下顺序创建：
1. 项目初始化 + 全局样式 + 布局组件
2. 路由配置 + 侧边栏菜单
3. 登录页
4. Dashboard 态势总览大屏（核心展示页）
5. 公共图表组件封装
6. 舆情监测模块
7. 深度分析模块
8. 传播追踪模块
9. 预警中心模块
10. 安全态势模块
11. 智能体管理中心

请先从第1-4步开始，把项目骨架和最重要的Dashboard大屏做出来。
```
